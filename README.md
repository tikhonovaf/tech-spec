# Тестовое задание
Есть 3 сервиса.
1. Реклама баннерная (видео баннер, изображение, текстовый)  - типы реклам
2. Курсы валют (код валюты, название, покупка, продажа) - создать таблицу и заполнение из ЦБ
3. Список услуг (название, описание, изображение) 
4. Услуга + реклама + цена в рублях

Есть клиентские приложения:
- ЖК панель, которая отображает рекламу и курсы. Обращается к сервисам 1 и 2.
- - Добавим цену в рублях.  И можно отображать цены в рублях и любой заданной валюте.
- 
- Терминал (киоск), который отображает меню услуг и выдает талончики. Может отображать курсы и рекламу. Обращается к сервису 1, 2, 3.

Необходимо:
1. Реализовать API для клиентских приложений.
2. *. Клиентские приложения (ЖК, Киоск) не должны отображать изменения, без подтверждения человека.
   Например: Мы изменили в сервисе рекламу, загрузили новую картинку или изменили существующую, она должна пройти модерацию человеком сначала, после подтверждения на ЖК будут видны изменения.
   Второй пример: Киоск отображает меню услуг, в адмике изменили название услуги, это название должна пройти модерацию, только после этого Киоск ее получит. До подтверждения, Киоск отображает старое название.


 
5. Реализация:
     Есть три категории пользователей
        - покупатель рекламы,  видит только прошедшее модерацию
        - сотрудние службы поддержки - видит только записи, на которые нет ссылок из отредактированных записе.
          может вносить изменения, но не может модерировать. После
          внесения измененений создается новая запись, содержащая ссылку на запись, которая редактировалась и измененные данные.  
        - модератор - может модерировать изменения. После модерация исходная запись удаляется. Данная запись становится покупателю. 

6. Реклама баннерная (видео баннер, изображение, текстовый)
   - выброрка категорий баннеров
   - выборка типов реклам с указанием категории
7. Курсы валют 
   - загружаются планировщиком из ЦБ
   - выборка сервисом
8. При добавлении услуги, она не подтверждена и не видна покупателю   
9. После подтверждения услуги, она видна покупателю
10. При  редактировании подтвержденной услуги создается новая услуга с измененными значениями, содержащая ссылку на редактируемую услугу
11. После подтверждения, услуга со ссылкой удаляется
